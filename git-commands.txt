=====================================
  GIT COMMANDS — PORTFOLIO CHEATSHEET
=====================================

All commands below must be run inside the D:/PORTO folder.
In Claude Code, prefix every command with  !  to run it.
Example:  ! git status


-------------------------------------
  DAILY WORKFLOW
-------------------------------------

Check what files you changed:
  git status

See the exact changes you made:
  git diff

Save + upload ALL changes to GitHub (do this after editing anything):
  git add -A && git commit -m "Update portfolio" && git push

Save + upload with a custom message (recommended):
  git add -A && git commit -m "Add new project: Project Name" && git push


-------------------------------------
  UPLOADING SPECIFIC FILES ONLY
-------------------------------------

Stage (prepare) a specific file:
  git add data/projects.js

Stage multiple specific files:
  git add data/projects.js index.html

Commit and push after staging:
  git commit -m "Your message here" && git push


-------------------------------------
  VIEWING HISTORY
-------------------------------------

See list of past commits (press Q to quit):
  git log --oneline

See full commit history with dates:
  git log


-------------------------------------
  UNDOING CHANGES
-------------------------------------

Undo changes to a file (revert to last saved version):
  git restore data/projects.js

Undo ALL unsaved changes (careful — cannot be undone!):
  git restore .


-------------------------------------
  SYNC & REMOTE
-------------------------------------

Download latest changes from GitHub (if edited online):
  git pull

Check remote URL (where the code is pushed to):
  git remote -v
